# BetterResidence MVP Todo

本文档由AI维护，记录 BetterResidence 当前 MVP 的完成度、仍在推进的核心任务，以及后续可以规划的扩展方向。内容以项目整体为范围，不只覆盖最近完成的权限系统改动。

本文档的存在意味着插件仍在MVP阶段。我会在插件进入Beta阶段时删除本文档。

## MVP 已完成

### 插件基础结构

项目已经具备 Paper 插件的基本生命周期、主命令注册、Gradle 构建配置和依赖接入。当前使用 Java 21、Paper API `1.21.8`，并通过 ShadowJar relocate PlanetLib，降低运行时依赖冲突风险。

### 配置与语言系统

插件已经支持 `config.yml`、语言文件、消息主题和 `/res reload`。权限组、目标组、语言和主题都可以在重载时重新加载，非法关键配置会在启动或重载时尽早暴露。

### 数据库与持久化结构

项目已经使用 SQLite 保存领地、区域、权限、玩家组、自定义组和 flag 等数据。区域坐标使用 SQLite R-tree 表存储，为后续高频位置查询和事件保护提供基础性能保障。

### 领地数据模型

`Claim` 已经能够表示领地所有者、名称、UUID、父领地、区域、权限、玩家组和 flag。父子领地、子领地列表、领地深度和基础 flag 读取逻辑也已经具备。

### 领地创建与查询核心

底层已经支持通过 `ClaimManager.createClaim(...)` 创建领地，并通过 `ClaimManager.findClaimAt(...)` 查询某个位置所在领地。查询逻辑会结合 R-tree、区块缓存和领地深度，在重叠区域中优先选择更深层的子领地。`/res create <名称>` 已经接入玩家选区、重叠校验、领地数量限制、创建价格和扣款流程。

### 玩家选区与创建可视化

玩家可以使用配置的选区工具设置两个角点。创建预览会根据可创建、受限制和冲突状态显示不同颜色；手持选区工具时也可以自动显示附近领地边界。缺少选区时的重复提示已经修复，避免玩家移动时刷屏。

### 权限组系统

权限组已经统一为 `ClaimGroup` 模型，支持配置默认组和每个领地的自定义组。系统已经区分真实权限组、`访客` 和 `任何人`，并支持 `-1000..1000` 的权重区间以及负权重黑名单场景。

### 玩家信任与权限组分配

`/res trust` 和 `/res untrust` 已经可以管理玩家在当前领地中的权限组。命令支持默认信任组、黑名单等配置组、自定义组，以及用 `访客` 移除玩家组记录。

### 权限规则系统

项目已经具备基于权重阈值的权限判断：玩家组权重大于等于权限要求时允许。权限规则支持默认配置、领地覆盖、全局通配符、命名空间通配符和具体权限优先级。

### 目标后缀与目标组

权限已经支持 `permission:target` 格式，例如 `block.break:oak_log` 或 `block.interact:container`。目标类型、目标组、目标组优先级和目标 tab complete 已经实现，并由 `target-groups.yml` 提供默认配置。

### 权限设置命令

`/res set <权限> <权限组>` 已经可以修改当前领地的权限阈值。命令会校验权限 ID、目标后缀和权限组，并提供权限名、目标和权限组补全，其中权限组补全已按权重从高到低排序。

### 命令权限保护

需要领地管理权限的命令已经统一继承 `ClaimAdminCommand`。`/res trust`、`/res untrust` 和 `/res set` 都要求执行者是玩家、位于领地内，并拥有当前领地的 `admin` 权限。

### 事件保护

当前已经接入 `BlockBreakEvent`，玩家在领地内破坏方块时会检查 `block.break` 权限。无权限时事件会被取消，并通过 actionbar 给出本地化提示。

### 进入/离开领地提示

插件已经监听玩家移动和传送，检测玩家进入或离开领地。进入/离开提示支持默认语言消息，也支持通过领地 flag 覆盖。

### 区域可视化基础

项目已经实现粒子区域框可视化，包括边框、面、渲染范围裁剪和玩家级任务管理。这套能力已经接入创建选区、附近领地显示，以及进入/离开领地时的短暂边界闪烁。

### 领地信息与列表命令

`/res info` 已经可以查看当前位置或指定领地的信息，包括所有者、父领地、深度、子领地、当前权限组、权限组数量和成员数量。`/res list` 已经可以分页列出自己或指定玩家的领地，并显示范围摘要和可交互的翻页入口。

### 领地删除

`/res remove` 已经支持删除当前领地，并通过带点击按钮的确认消息避免误删。删除会清理领地自身的区域、权限、成员、自定义组和 flag 数据；直接子领地会自动升级到被删除领地的父领地，顶级领地被删除时直接子领地会升级为顶级领地。

### 文档

已经编写 `docs/permission-system.md`，说明权限权重、访客、任何人、黑名单、命令、目标后缀和规则优先级。该文档可以作为后续用户文档和配置文档的基础。

## MVP 进行中 / 仍需完成

### 移除测试代码

当前启动流程仍会创建测试领地，玩家加入时也会显示测试区域可视化。发布 MVP 前应移除这些硬编码测试逻辑，或改为明确的 debug 配置开关。

### 领地删除和区域管理

当前已经支持删除领地，但还缺少调整区域、追加区域和移除区域的命令。实现时需要同时处理数据库清理、缓存失效、子领地关系和可能的经济退款策略。

### 权限保护事件覆盖

当前只保护方块破坏，事件覆盖还不足以用于真实服务器。MVP 至少需要继续接入放置方块、方块交互、实体攻击、实体交互，以及常见绕过路径的基础保护。

### 权限组管理命令

数据库已经支持每个领地的自定义权限组，但玩家还不能通过命令创建、删除、重命名或调整权重。后续需要补齐 `/res group ...` 系列命令，并处理名称冲突和删除组后的玩家迁移策略。

### 领地 flag 管理命令

`claim_flags` 底层已经存在，但还没有 `/res flag` 命令。后续需要定义 flag 注册表、类型校验、默认值、补全和用户可配置项，例如进入消息、离开消息和环境行为开关。

### 可视化与选区体验

创建选区、附近领地显示和进入/离开边界闪烁已经接入粒子可视化。后续还需要提供显式的显示/隐藏命令，支持查看指定领地、编辑区域时的预览，以及在不手持选区工具时更可控地切换显示目标。

### 经济与限制

创建领地已经接入 Vault 余额检查与按体积扣款，也支持玩家顶级领地数量限制。后续还需要补齐扩展计费、删除退款、面积/体积上下限、不同世界价格倍率和管理员豁免策略。

### 缓存与一致性

项目已有 Claim 缓存、区域查询缓存、权限节点缓存和玩家组缓存，但修改路径还没有完全统一。随着更多命令落地，需要明确每类数据变更后的缓存失效策略。

### 配置校验与默认值

权限组配置已经有较完整校验，部分不适合作为用户配置的选项已经改为硬编码。默认权限、目标组、自定义组写入和可视化配置仍需要补充校验；默认权限策略也需要最终确认，例如默认是否允许 trusted 拥有大部分行为权限。

### 测试与质量保障

当前没有自动化测试源码，主要依赖构建和手动验证。MVP 前建议优先补权限匹配、目标匹配、空间查询、命令行为和核心事件保护的测试或手动回归清单。

## 未来计划（也在MVP实现范围内，但是没那么紧急）

### 命令与用户体验

可以继续扩展 `/res help`、`/res select`、`/res rename`、`/res tp`、`/res setowner` 和子领地命令。后续也可以加入聊天 hover、click suggestion、分页列表和更友好的确认流程。

### 权限系统扩展

可以增加更多内置权限，例如物品使用、拾取、丢弃、载具、传送、飞行、红石和特殊交互。也可以加入权限模板、批量设置、权限解释和 debug 命令，降低配置复杂度。

### 领地生命周期与管理

后续可以支持领地转让、合并、拆分、扩展、收缩、过期和长期未登录清理。管理员侧也需要更完整的强制管理、绕过和审计能力。

### 生态兼容

PlaceholderAPI 当前只是软依赖，后续可以提供当前领地、所有者、权限组等占位符。也可以开放 API，让其他插件注册权限、目标类型、flag、事件监听和额外限制规则。

### 性能与可观测性

可以增加空间查询缓存配置、慢查询日志、权限检查 debug 统计和保护事件性能采样。大型服务器场景还需要压力测试文档和推荐配置。

### 文档与发布

后续需要补安装文档、配置参考、命令参考、玩家教程、FAQ、升级说明和发布前 checklist。发布前尤其需要确认测试代码移除、默认配置安全性、构建产物和真实服务器 smoke test。
