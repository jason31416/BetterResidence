# BetterResidence MVP Todo

本文档由AI维护，记录 BetterResidence 当前 MVP 的完成度、仍在推进的核心任务，以及后续可以规划的扩展方向。内容以项目整体为范围，不只覆盖最近完成的权限系统改动。

本文档的存在意味着插件仍在MVP阶段。我会在插件进入Beta阶段时删除本文档。

## MVP 已完成

### 插件基础结构

项目已经具备 Paper 插件的基本生命周期、主命令注册、Gradle 构建配置和依赖接入。当前使用 Java 21、Paper API `1.21.8`，并通过 ShadowJar relocate PlanetLib，降低运行时依赖冲突风险。

### 配置与语言系统

插件已经支持 `config.yml`、语言文件、消息主题和 `/res reload`。权限组、目标组、语言和主题都可以在重载时重新加载，非法关键配置会在启动或重载时尽早暴露。

### 数据库与持久化结构

项目已经使用 SQLite 保存领地、区域、权限、玩家组、自定义组和 flag 等数据。区域坐标使用 SQLite R-tree 表存储，为后续高频位置查询和事件保护提供基础性能保障。

### 领地数据模型

`Claim` 已经能够表示领地所有者、名称、UUID、父领地、区域、权限、玩家组和 flag。父子领地、子领地列表、领地深度、基础 flag 读取逻辑，以及基于父领地多区域并集的子领地覆盖校验也已经具备。

### 领地创建与查询核心

底层已经支持通过 `ClaimManager.createClaim(...)` 创建领地，并通过 `ClaimManager.findClaimAt(...)` 查询某个位置所在领地。查询逻辑会结合 R-tree、区块缓存和领地深度，在重叠区域中优先选择更深层的子领地。`/res create <名称>` 已经接入玩家选区、重叠校验、顶级领地数量限制、创建价格和扣款流程。

`/res create <名称>` 也已经支持创建子领地。子领地会自动选择最深的完全覆盖选区的领地作为父领地，不扣款、不计入玩家顶级领地数量，并要求执行者在父领地内拥有 `admin.subclaim` 权限。子领地创建时会复制父领地当前 flags，但不会继承权限、成员或自定义权限组。

### 玩家选区与创建可视化

玩家可以使用配置的选区工具设置两个角点。创建预览会根据可创建、受限制和冲突状态显示不同颜色；手持选区工具时也可以自动显示附近领地边界。缺少选区时的重复提示已经修复，避免玩家移动时刷屏。

### 权限组系统

权限组已经统一为 `ClaimGroup` 模型，支持配置默认组和每个领地的自定义组。系统已经区分真实权限组、`访客` 和 `任何人`，并支持 `-1000..1000` 的权重区间以及负权重黑名单场景。

### 玩家信任与权限组分配

`/res trust` 和 `/res untrust` 已经可以管理玩家在当前领地中的权限组。命令支持默认信任组、黑名单等配置组、自定义组，以及用 `访客` 移除玩家组记录。

### 权限规则系统

项目已经具备基于权重阈值的权限判断：玩家组权重大于等于权限要求时允许。权限规则支持默认配置、领地覆盖、全局通配符、命名空间通配符、具体权限优先级和层级继承。

### 目标后缀与目标组

权限已经支持 `permission:target` 格式，例如 `block.break:oak_log` 或 `block.interact:container`。目标类型、目标组、目标组优先级和目标 tab complete 已经实现，并由 `target-groups.yml` 提供默认配置。

### 权限设置命令

`/res set <权限> <权限组>` 已经可以修改当前领地的权限阈值。命令会校验权限 ID、目标后缀和权限组，并提供权限名、目标和权限组补全，其中权限组补全已按权重从高到低排序。

### 命令权限保护

需要领地管理权限的命令已经改为检查具体的 `admin.xxx` 权限。`/res trust` 和 `/res untrust` 使用 `admin.trust`，`/res set` 使用 `admin.setpermission`，`/res flag` 和 `/res settp` 使用 `admin.setflag`，区域追加、删除、查看和扩缩分别检查对应的 `admin.area.xxx` 权限。

### 事件保护

当前已经接入主要玩家行为事件保护，包括方块破坏、放置、桶操作、方块交互、物品使用、实体伤害、实体交互、载具交互、悬挂实体、刷怪蛋/载具/盔甲架生成、投射物使用、进入领地、飞行滑翔、物品丢弃和物品拾取。无权限时事件会被取消，并通过 actionbar 给出本地化提示。

物品丢弃和拾取已拆分为 `dropitem.throw` 与 `dropitem.pickup`，并支持材料目标后缀和 `dropitem` 层级权限。默认配置中访客允许 `dropitem`，因此新领地默认允许访客丢弃和拾取物品。

环境类保护也已经通过 flag 覆盖常见非玩家路径，包括水/岩浆流动、活塞跨领地移动、爆炸、火焰、实体改方块、作物和自然生长、腐烂、方块形成、传送门生成，以及自然/刷怪笼生物生成。

### 领地 flag 系统

已经实现 `FlagRegistry`、flag 类型校验、层级 flag、默认值、`/res flag <标志> [值]` 和 `/res flaglist`。当前内置 flag 包括进入/离开消息、传送点、时间、天气、流体、活塞、爆炸、火焰、实体改方块、生长、腐烂、方块形成、传送门生成和生物生成等。

### 领地传送

`/res settp` 已经可以把当前领地传送点保存到 `teleport-location` flag，并校验传送点必须在领地内。`/res tp <领地>` 已经支持权限检查、传送点有效性校验、可配置 warmup、移动取消和异步传送结果提示。

### 进入/离开领地提示

插件已经监听玩家移动和传送，检测玩家进入或离开领地。进入/离开提示支持默认语言消息，也支持通过领地 flag 覆盖。

### 区域可视化基础

项目已经实现粒子区域框可视化，包括边框、面、渲染范围裁剪和玩家级任务管理。这套能力已经接入创建选区、附近领地显示，以及进入/离开领地时的短暂边界闪烁。

### 领地信息与列表命令

`/res info` 已经可以查看当前位置或指定领地的信息，包括所有者、父级链式路径、直接子领地列表、当前权限组、权限组数量和成员数量。子领地列表中的条目可以点击进入对应子领地的 `/res info`。`/res list` 已经可以分页列出自己或指定玩家的领地，并显示范围摘要和可交互的翻页入口。

### 领地删除与区域管理

`/res remove` 已经支持删除当前领地，并通过带点击按钮的确认消息避免误删。删除会清理领地自身的区域、权限、成员、自定义组和 flag 数据；直接子领地会自动升级到被删除领地的父领地，顶级领地被删除时直接子领地会升级为顶级领地。

`/res area` 已经支持向当前或指定领地追加区域、按当前位置删除所在区域，以及查看当前领地区域列表。顶级领地追加区域会按方块数扣款，并且不能与任何已有区域重叠；子领地追加区域必须完全位于直接父领地范围内，允许与父级链重叠且免费，但仍不能与自身、兄弟、后代或无关领地重叠。删除区域不会暴露内部 `area_id`，且禁止删除最后一个区域或包含子领地的区域。

`/res expand <方块数>` 和 `/res contract <方块数>` 已经支持按玩家视角方向扩展或收缩当前位置所在区域。顶级领地扩展会按新增方块数扣款；子领地扩缩需要保持在父领地覆盖范围内；收缩会校验后代领地区域仍被当前领地覆盖。

### 文档

已经编写 `docs/permission-system.md`，说明权限权重、访客、任何人、黑名单、命令、目标后缀和规则优先级。该文档可以作为后续用户文档和配置文档的基础。

## MVP 进行中 / 仍需完成

### 高级区域管理收尾

当前已经支持追加、删除、扩展和收缩区域，但还缺少区域合并、拆分、指定区域的显式可视化和删除退款策略。实现时需要继续处理缓存失效、子领地关系和经济差价策略。

### 权限组管理命令

数据库已经支持每个领地的自定义权限组，但玩家还不能通过命令创建、删除、重命名或调整权重。后续需要补齐 `/res group ...` 系列命令，并处理名称冲突和删除组后的玩家迁移策略。

### 领地 flag 管理命令

基础 `/res flag` 和 `/res flaglist` 已经实现。后续仍需要补充更友好的 flag 文档、分组展示、批量重置、默认值说明，以及面向普通玩家的解释文本。

## 未来计划（也在MVP实现范围内，但是没那么紧急）

### 命令与用户体验

可以继续扩展 `/res help`、`/res rename`、`/res setowner` 和子领地命令。`/res tp` 和 `/res settp` 已经具备基础能力，但后续仍可以增加公开领地传送、收藏、历史记录或更友好的补全。后续也可以加入更多聊天 hover、click suggestion、分页列表和更友好的确认流程。

### 领地生命周期与管理

后续可以支持领地转让、合并、拆分、扩展、收缩、过期和长期未登录清理。管理员侧也需要更完整的强制管理、绕过和审计能力。

### 生态兼容

PlaceholderAPI 当前只是软依赖，后续可以提供当前领地、所有者、权限组等占位符。也可以开放 API，让其他插件注册权限、目标类型、flag、事件监听和额外限制规则。

### 性能与可观测性

可以增加空间查询缓存配置、慢查询日志、权限检查 debug 统计和保护事件性能采样。大型服务器场景还需要压力测试文档和推荐配置。

### 文档与发布

后续需要补安装文档、配置参考、命令参考、玩家教程、FAQ、升级说明和发布前 checklist。发布前尤其需要确认测试代码移除、默认配置安全性、构建产物和真实服务器 smoke test。
